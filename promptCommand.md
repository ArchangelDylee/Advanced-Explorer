## 1. 프로그램이 뭘 하는 건가요?

### 어떤 프로그램인가요?
```
"Advanced Explorer"는 컴퓨터에 있는 파일들을 쉽게 찾고 관리하는 프로그램이에요.
마치 구글 검색처럼 파일 안에 있는 내용까지 검색할 수 있어요!

주요 기능:
1. 📁 파일 탐색 - 폴더와 파일을 쉽게 찾아다닐 수 있어요
2. 🔍 똑똑한 검색 - 파일 이름뿐만 아니라 파일 안의 내용도 찾을 수 있어요
3. 📝 자동 요약 - 긴 문서를 짧게 요약해줘요
4. 🎨 예쁜 디자인 - 다크 모드로 눈이 편해요
```

## 2. 프로그램 만들기 시작

### 2-1. 필요한 프로그램 설치하기

**1단계: 기본 프로그램들 설치**
```
다음 프로그램들을 컴퓨터에 먼저 설치해주세요:

1. Node.js - 웹 기술로 프로그램을 만드는 도구
   (www.nodejs.org에서 다운로드)
   
2. Python - 파일 내용을 읽고 분석하는 도구
   (www.python.org에서 다운로드, 3.11 버전)
   
3. Visual Studio Code - 코드를 작성하는 편집기
   (code.visualstudio.com에서 다운로드)
```

**2단계: 프로젝트 폴더 만들기**
```
1. 바탕화면에 "Advanced Explorer"라는 폴더를 만드세요
2. 그 폴더를 Visual Studio Code로 열으세요
3. 아래 명령어를 터미널에 입력하세요:

npm init -y
npm install react react-dom
npm install electron
npm install vite
```

## 3. 화면 꾸미기

### 3-1. 프로그램 화면 구성하기

**왼쪽 사이드바 만들기**
```
왼쪽에는 이런 것들이 보여야 해요:

1. 즐겨찾기 (자주 가는 폴더들)
   - 문서 폴더
   - 바탕화면
   - 다운로드
   - 사진
   - 음악

2. 폴더 트리 (컴퓨터의 모든 폴더들)
   - C 드라이브
   - 사용자 폴더들
   
💡 즐겨찾기와 폴더 트리 사이의 선을 마우스로 드래그하면
   높이를 조절할 수 있어요!
```

**가운데 파일 목록 만들기**
```
가운데에는 파일들이 표처럼 보여야 해요:

- 파일 이름 (정렬 가능)
- 파일 크기 (정렬 가능)
- 수정 날짜 (정렬 가능)

파일 옆에 작은 표시가 보여요:
- ✓ (초록색) = 이 파일은 검색할 수 있어요
- ○ (회색) = 아직 검색 준비가 안 됐어요
```

**오른쪽 패널 만들기**
```
오른쪽에는 4개의 탭이 있어요:

1. 📄 내용 보기 - 파일 내용을 보여줘요
2. 🔍 검색 로그 - 검색 결과를 보여줘요
3. 📊 인덱싱 로그 - 파일 처리 상황을 보여줘요
4. 🗂️ 색인 관리 - 검색 데이터베이스 관리
```

### 3-2. 색상 정하기

**다크 모드 색상**
```
프로그램을 멋지게 만들려면 이런 색상을 사용하세요:

- 배경색: 검은색 계열 (#191919)
- 패널: 조금 밝은 검은색 (#202020)
- 글자: 흰색 (#FFFFFF)
- 강조색: 파란색 (#0067C0) - Windows 스타일!
- 테두리: 회색 (#444444)

이렇게 하면 Windows 11처럼 보여요!
```

## 4. 파일 찾기 기능

### 4-1. 파일 탐색하기

**폴더 이동하기**
```
사용자가 폴더를 더블클릭하면:
1. 그 폴더 안의 파일들을 가져와요
2. 파일 목록을 화면에 보여줘요
3. 주소창에 현재 위치를 표시해요

뒤로/앞으로 버튼도 만들어서:
- 이전 폴더로 갈 수 있게 해요
- 다음 폴더로 갈 수 있게 해요
```

**파일 정렬하기**
```
파일 목록 위의 제목을 클릭하면:
- 이름 순으로 정렬 (가나다순, ABC순)
- 크기 순으로 정렬 (작은 것부터, 큰 것부터)
- 날짜 순으로 정렬 (오래된 것부터, 최신순)

폴더는 항상 파일보다 위에 있어야 해요!
```

### 4-2. 여러 탭 사용하기

**탭 만들기**
```
여러 폴더를 동시에 볼 수 있도록:
1. + 버튼을 눌러서 새 탭을 만들어요
2. 각 탭마다 다른 폴더를 열 수 있어요
3. 탭 이름은 폴더 이름으로 자동으로 바뀌어요
4. X 버튼으로 탭을 닫을 수 있어요

프로그램을 종료해도 탭들이 저장되어 있어요!
```

## 5. 파일 내용 검색

### 5-1. 검색 준비하기 (인덱싱)

**인덱싱이 뭔가요?**
```
인덱싱은 책의 "찾아보기"를 만드는 것과 같아요.
파일 안의 모든 단어를 미리 정리해놓으면
나중에 빠르게 찾을 수 있어요!

처리할 수 있는 파일:
- PDF 파일
- 워드 문서 (DOCX, DOC)
- 파워포인트 (PPTX, PPT)
- 엑셀 (XLSX, XLS)
- 텍스트 파일 (TXT, MD)
- 한글 문서 (HWP)
```

**인덱싱 시작하기**
```
1. 검색하고 싶은 폴더를 선택하세요
2. "색인 시작" 버튼을 누르세요
3. 프로그램이 자동으로 파일들을 읽기 시작해요

진행 상황을 볼 수 있어요:
- 처리중: 파일명, 크기, 글자 수
- 이전완료: 이미 처리했던 파일은 건너뛰어요
- ✓ DB완료: 데이터베이스에 저장 완료!
```

**똑똑한 기능**
```
마우스나 키보드를 사용하면:
- 프로그램이 자동으로 멈춰요 (컴퓨터가 느려지지 않게!)
- 2초 동안 가만히 있으면 다시 시작해요

파일이 바뀌지 않으면:
- 다시 처리하지 않아요 (빠르게!)
- "이전 처리 완료"라고 표시해요
```

### 5-2. 검색하기

**검색창 사용하기**
```
위쪽 검색창에 단어를 입력하세요:

1. 파일명 검색
   - "보고서"를 입력하면 "보고서"가 들어간 파일명을 찾아요
   
2. 내용 검색 (체크박스 켜기)
   - "프로젝트"를 입력하면 파일 안에 "프로젝트"가 있는 파일을 찾아요
   
3. 하위 폴더 검색 (체크박스 켜기)
   - 현재 폴더뿐만 아니라 안에 있는 모든 폴더도 검색해요
```

**검색 방법**
```
1. 단어 여러 개 (AND 조건)
   입력: microsoft SKP
   결과: "microsoft"와 "SKP" 둘 다 들어간 파일
   
2. 정확한 문장 (특수문자 포함)
   입력: "microsoft & SKP"
   결과: 정확히 "microsoft & SKP"가 들어간 파일
   
💡 팁: 특수문자(&, @, # 등)가 있으면 따옴표를 꼭 쓰세요!
```

**검색 결과 보기**
```
검색하면 이렇게 보여요:

🔍 검색 시작: "프로젝트" (C:\Users\Documents)
   파일명 매칭: 총 3개 발견
   내용 매칭: 총 12개 발견
✓ 검색 완료: 총 15개 파일 (소요시간: 0.5초)

- 파일명 매칭: 프로젝트 계획서.docx
- 내용 5개 매칭: 회의록_2023.docx
- 내용 3개 매칭: 업무 보고서.pdf
```

## 6. 요약 기능

### 6-1. 긴 문서를 짧게 만들기

**요약이 어떻게 작동하나요?**
```
TextRank라는 똑똑한 방법을 사용해요:

1. 문서의 모든 문장을 읽어요
2. 어떤 문장이 중요한지 점수를 매겨요
3. 점수가 높은 문장 5개를 골라요
4. 그 문장들을 순서대로 보여줘요

모든 언어(한글, 영어 등)에서 작동해요!
```

**요약 버튼 사용하기**
```
1. 파일을 선택하세요
2. 오른쪽 "내용 보기" 탭으로 가세요
3. "요약 생성" 버튼을 누르세요
4. 잠시 기다리면 요약이 나와요!

요약 결과에는:
- 원본 길이: 5028자
- 요약 길이: 1335자
- 압축률: 26.5%

문장 사이에 빈 줄이 있어서 읽기 편해요!
```

## 7. 기타 편리한 기능

### 7-1. 우클릭 메뉴

**파일이나 폴더를 우클릭하면**
```
작은 메뉴가 나타나요:

1. 📋 경로 복사
   - 파일이 있는 위치를 복사해요
   - 예: C:\Users\dylee\Documents\보고서.docx
   
2. 📝 이름 복사
   - 파일 이름만 복사해요
   - 예: 보고서.docx

복사한 내용은 다른 곳에 붙여넣기 할 수 있어요!
```

### 7-2. 파일 관리하기

**파일/폴더 작업**
```
위쪽 버튼들로 파일을 관리할 수 있어요:

📁 새 폴더
- 새로운 폴더를 만들어요

📋 복사
- 선택한 파일을 복사해요

✏️ 이름 변경
- 파일/폴더 이름을 바꿔요

🗑️ 삭제
- 파일/폴더를 휴지통으로 보내요

📌 붙여넣기
- 복사한 파일을 현재 폴더에 넣어요
```

### 7-3. 파일 타입 필터

**특정 종류의 파일만 보기**
```
체크박스로 보고 싶은 파일만 선택할 수 있어요:

✓ PPT - 파워포인트 파일
✓ DOC - 워드 문서
✓ HWP - 한글 문서
✓ TXT - 텍스트 파일
✓ PDF - PDF 파일
✓ CSV - 엑셀 파일

체크를 해제하면 그 종류의 파일은 안 보여요!
```

### 7-4. 즐겨찾기/폴더 트리 높이 조절

**화면을 내 맘대로 조절하기**
```
왼쪽 사이드바에서:

1. 즐겨찾기와 폴더 트리 사이에 얇은 선이 있어요
2. 그 선에 마우스를 올리면 커서가 바뀌어요
3. 마우스로 선을 위아래로 드래그하세요
4. 즐겨찾기 높이를 원하는 대로 조절할 수 있어요!

프로그램을 껐다 켜도 크기가 그대로 유지돼요!
```

### 7-5. 초기 화면 설정

**프로그램을 켜면 자동으로**
```
문서 폴더가 열려 있어요!

경로: C:\Users\[사용자이름]\Documents

가장 자주 사용하는 폴더라서 
자동으로 열리도록 했어요.
```

### 7-6. 접근 권한 관리

**볼 수 없는 파일은 숨기기**
```
컴퓨터에는 시스템 파일처럼 
일반 사용자가 볼 수 없는 파일들이 있어요.

프로그램이 자동으로:
- 읽을 수 없는 파일은 목록에 안 보여줘요
- 접근 권한이 없는 폴더는 숨겨요
- 오류 없이 깔끔하게 작동해요!
```

## 8. 실행 방법

### 8-1. 개발 모드로 실행하기

**터미널을 3개 열어서 각각 실행하세요**

```
터미널 1 (Python 백엔드):
cd python-backend
python server.py

→ 파일을 읽고 검색하는 부분이에요

터미널 2 (프론트엔드):
npm run dev

→ 화면을 만드는 부분이에요

터미널 3 (Electron):
npm run electron

→ 프로그램 창을 띄우는 부분이에요
```

**모든 게 정상이면**
```
✅ Python 서버: Running on http://127.0.0.1:5000
✅ Vite 서버: Local: http://localhost:5173
✅ Electron: 프로그램 창이 열려요!
```

## 9. 자주 묻는 질문

### Q1: 검색이 안 돼요!
```
A: 먼저 인덱싱을 해야 해요!

1. 검색하고 싶은 폴더를 선택하세요
2. "색인 시작" 버튼을 누르세요
3. 인덱싱이 끝날 때까지 기다리세요
4. 파일 옆에 ✓ 표시가 나타나면 검색할 수 있어요!
```

### Q2: 특수문자가 들어간 단어를 못 찾아요!
```
A: 따옴표를 사용하세요!

잘못된 예: microsoft & SKP
올바른 예: "microsoft & SKP"

특수문자(&, @, #, $ 등)가 있으면 
반드시 따옴표로 감싸주세요!
```

### Q3: 요약이 실패했어요!
```
A: 이런 것들을 확인해보세요:

1. 파일이 텍스트가 있는 파일인가요?
   (이미지만 있는 PDF는 요약이 안 돼요)
   
2. 파일이 너무 짧지 않나요?
   (최소 3문장은 있어야 해요)
   
3. Python 서버가 실행 중인가요?
   (터미널에서 확인하세요)
```

### Q4: 프로그램이 느려요!
```
A: 이렇게 해보세요:

1. 인덱싱을 잠시 멈추세요 (색인 중지 버튼)
2. 파일 타입 필터로 필요한 파일만 보세요
3. 너무 많은 탭을 열지 마세요 (3-4개 정도)
4. 검색 범위를 줄이세요 (하위 폴더 검색 끄기)
```

### Q5: 파일을 클릭했는데 내용이 안 보여요!
```
A: 확인해보세요:

1. 파일 옆에 ✓ 표시가 있나요?
   ○ 표시면 아직 인덱싱이 안 된 거예요
   
2. 오른쪽에서 "내용 보기" 탭을 선택했나요?

3. Python 서버가 실행 중인가요?
```

## 💡 더 알아두면 좋은 것

### 데이터는 어디에 저장되나요?
```
1. 검색 데이터베이스
   위치: python-backend/file_index.db
   내용: 파일 안의 모든 단어들
   
2. 설정 저장
   위치: 브라우저의 localStorage
   내용: 탭, 창 크기, 필터 설정 등
   
3. 로그 파일
   위치: python-backend/indexing_combined.log
   내용: 인덱싱 기록
```

### 프로그램을 친구에게 줄 수 있나요?
```
네! 하지만 실행 파일로 만들어야 해요.

electron-builder라는 도구를 사용하면
.exe 파일로 만들 수 있어요!

그러면 다른 컴퓨터에서도
설치만 하면 바로 사용할 수 있어요.
```

