## 1. 검색 관련 문제

### 문제 #1: 검색 결과에 잘못된 표시가 나와요

**무슨 문제였나요?**
```
파일 안에 "프로젝트"라는 단어가 있어서 찾았는데
검색 로그에는 "파일명 매칭"이라고 잘못 나왔어요.
실제로는 "내용 매칭"이 맞는데 말이에요!
```

**어떻게 고쳤나요?**
```
검색 결과에는 "어디서 찾았는지" 정보가 있어요.
- 파일 시스템에서 찾음 → "파일명 매칭"
- 데이터베이스에서 찾음 → "내용 매칭"

이 정보를 제대로 사용하도록 고쳤어요!
```

---

### 문제 #2: 특수문자가 들어간 단어를 못 찾아요

**무슨 문제였나요?**
```
"microsoft & SKP"를 검색하면 아무것도 안 나왔어요.
파일 안에 분명히 있는데도요!

원인: & 같은 특수문자를 검색 프로그램이 이해하지 못했어요.
```

**어떻게 고쳤나요?**
```
1. 따옴표로 감싼 검색 ("microsoft & SKP")이면
   특수문자가 있는지 확인해요
   
2. 특수문자가 있으면 LIKE 검색을 사용해요
   (정확히 그 문자를 찾는 방법)
   
3. 특수문자가 없으면 FTS5 검색을 사용해요
   (빠른 검색 방법)
```

**이제 이렇게 검색하세요:**
```
❌ 잘못된 방법: microsoft & SKP
✅ 올바른 방법: "microsoft & SKP"

특수문자(&, @, #, $ 등)가 있으면 따옴표를 꼭 쓰세요!
```

## 2. 파일 표시 문제

### 문제 #3: 인덱싱했는데 내용이 안 보여요

**무슨 문제였나요?**
```
1. 파일을 클릭했어요
2. 처음에는 내용이 보였어요
3. 다른 파일 클릭했다가 다시 돌아오면
4. "인덱싱 안됨" 메시지가 나왔어요!
```

**왜 그랬을까요?**
```
파일을 선택했을 때 내용을 불러오는 프로그램이
"파일이 바뀌었는지" 제대로 확인하지 못했어요.

같은 파일을 다시 선택해도 
"이전 파일과 같네" 하고 무시해버렸어요.
```

**어떻게 고쳤나요?**
```
파일의 "경로"(위치)를 확인하도록 바꿨어요.
경로가 바뀌면 다른 파일이니까
내용을 다시 불러오게 했어요!
```

---

### 문제 #4: 파일 옆에 ✓ 표시가 나오는데 내용이 없어요

**무슨 문제였나요?**
```
파일 목록에는 ✓ (인덱싱 완료) 표시가 있는데
파일을 클릭하면 "인덱싱 안됨"이라고 나왔어요.
```

**어떻게 고쳤나요?**
```
1. 파일 상태를 확인하는 기능을 만들었어요
   (이 파일이 인덱싱됐나요? → 네/아니오)
   
2. 폴더를 열 때마다 모든 파일의 상태를 확인해요
   
3. 상태에 맞게 ✓ 또는 ○ 표시를 해줘요
```

---

### 문제 #5: 권한 없는 파일도 목록에 나와요

**무슨 문제였나요?**
```
시스템 폴더 같은 곳에 들어가면
볼 수 없는 파일들이 목록에 나왔어요.
클릭하면 오류가 났어요!
```

**어떻게 고쳤나요?**
```
파일 목록을 보여주기 전에:
1. 이 파일을 읽을 수 있나요? 확인해요
2. 읽을 수 없으면 목록에 넣지 않아요
3. 오류 없이 깔끔하게 작동해요!
```

## 3. 데이터베이스 문제

### 문제 #6: 파일을 처리해도 DB에 저장이 안 돼요

**무슨 문제였나요?**
```
파일 내용을 읽어서 처리했는데
데이터베이스에 저장이 안 됐어요.
나중에 검색하려고 하면 아무것도 없었어요!
```

**왜 그랬을까요?**
```
데이터베이스에 "저장 완료!" 명령을 안 했어요.
마치 편지를 쓰고 봉투에 넣었는데
우체통에 넣지 않은 것과 같아요.
```

**어떻게 고쳤나요?**
```
1. WAL 모드를 켰어요
   (여러 사람이 동시에 쓸 수 있게)
   
2. 작업할 때마다 명령을 명확하게 해요:
   - "시작!" (BEGIN)
   - 파일 정보 저장
   - "완료!" (COMMIT)
   
3. 오류가 나면 취소해요 (ROLLBACK)
```

## 4. 요약 기능 문제

### 문제 #7: 요약 버튼을 누르면 "실패" 나와요

**무슨 문제였나요?**
```
"요약 생성" 버튼을 누르면
"요약 실패" 메시지만 나왔어요.
```

**첫 번째 원인: summarizer가 없어요**
```
요약을 만드는 도구(summarizer)를 
프로그램이 찾지 못했어요.

해결: 
summarizer를 프로그램 시작할 때 
미리 만들어두도록 고쳤어요!
```

**두 번째 원인: 한글 처리 문제**
```
한글을 처리하려면 konlpy라는 도구가 필요한데
이게 설치하기 너무 복잡했어요.
(Java도 설치해야 하고...)

해결:
영어 처리 방식으로 한글도 처리하도록 바꿨어요.
요약은 문장의 중요도만 보면 되니까 괜찮아요!
```

**세 번째 원인: numpy가 없어요**
```
numpy라는 숫자 계산 도구가 없었어요.

해결:
requirements.txt 파일에 numpy를 추가했어요.
pip install numpy로 설치하면 돼요!
```

## 5. 화면 조절 문제

### 문제 #8: 즐겨찾기와 폴더 트리 높이를 못 바꿔요

**무슨 문제였나요?**
```
즐겨찾기가 너무 높아요!
폴더 트리가 너무 작아요!
하지만 높이를 조절할 수가 없었어요.
```

**어떻게 고쳤나요?**
```
1. 즐겨찾기와 폴더 트리 사이에 선을 그었어요
2. 그 선을 마우스로 잡아당길 수 있게 했어요
3. 위아래로 드래그하면 높이가 바뀌어요!
4. 최소 50px까지 줄일 수 있어요
5. 설정이 자동으로 저장돼요
```

## 6. 프로그램 실행 문제

### 문제 #9: 프로그램이 느려요

**해결 방법:**
```
1. 인덱싱 중지하기
   - "색인 중지" 버튼 누르기
   
2. 파일 타입 필터 사용하기
   - 필요한 파일만 체크하기
   - 예: PDF, 워드만 보기
   
3. 탭 정리하기
   - 탭을 3-4개만 열기
   - 안 쓰는 탭은 닫기
   
4. 검색 범위 줄이기
   - 하위 폴더 검색 끄기
   - 특정 폴더만 검색하기
```

---

### 문제 #10: 프로그램 재시작하는 방법

**모든 프로세스 종료하고 재시작:**
```
1단계: 모든 프로세스 종료
Get-Process python,node,electron -ErrorAction SilentlyContinue | Stop-Process -Force
Start-Sleep -Seconds 2

2단계: Python 백엔드 시작 (새 터미널)
cd "python-backend"
python server.py

3단계: Vite 서버 시작 (새 터미널)
npm run dev

4단계: Electron 앱 시작 (새 터미널)
Start-Sleep -Seconds 5
npm run electron
```

## 💡 디버깅 팁

### 문제가 생기면 이렇게 확인하세요

**1단계: 로그 확인하기**
```
Python 터미널을 보세요:
- "Running on http://127.0.0.1:5000" 나오나요?
- 오류 메시지는 없나요?
```

**2단계: 브라우저 콘솔 확인하기**
```
F12를 누르고 Console 탭을 보세요:
- 빨간 오류 메시지가 있나요?
- 무슨 오류인가요?
```

**3단계: 네트워크 확인하기**
```
F12 → Network 탭을 보세요:
- API 호출이 200 (성공)인가요?
- 404 (못찾음) 또는 500 (오류)인가요?
```

## 📚 추가 도움말

### Python 관련 오류
```
"ModuleNotFoundError" 나오면:
→ pip install [모듈이름]

"Permission denied" 나오면:
→ 관리자 권한으로 실행

"Port already in use" 나오면:
→ 이미 실행 중인 프로세스 종료
```

### Node.js 관련 오류
```
"Cannot find module" 나오면:
→ npm install

"Port 5173 is in use" 나오면:
→ Vite 서버 종료하고 재시작

"EACCES" 나오면:
→ 폴더 권한 확인
```
